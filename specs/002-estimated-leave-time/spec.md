## 목적

일평균 필요 근무시간을 머릿속으로 역산해서 "몇 시에 퇴근할 수 있지?"를 계산하는 게 번거롭다.
출근 시간 + 일평균 필요 근무시간 + 점심시간을 자동 계산해서 **"오늘 18:32 퇴근 가능"** 을 바로 보여준다.

---

## 입력 / 출력

### 입력
| 필드 | 타입 | 소스 | 확인 | 규칙 |
|------|------|------|:----:|------|
| 출근 시간 | HH:MM | 네이버 웍스 홈 "출퇴근" 위젯 크롤링 | [x] | "출근 09:53" 텍스트에서 추출. 스크린샷 확인 완료 |
| 일평균 필요 근무시간 | number (분) | 기존 calcRemainingInfo() 결과 | [x] | 001 피처에서 이미 계산 중 |
| 점심시간 | 1시간 고정 | 하드코딩 | [x] | 12:00~13:00, 근무시간에서 제외 |
| 코어타임 | 10:00~16:00 | 하드코딩 | [x] | 최소 퇴근 가능 시간 = 16:00 |

### 출력 (팝업 UI)
- 성공: **"오늘 18:32 퇴근 가능"** (강조 카드 아래에 표시)
- 출근 전: "출근 기록이 없습니다" (회색 텍스트)
- 이미 퇴근 가능: **"지금 퇴근 가능!"** (초록색)

---

## 계산 로직

```
1. 퇴근 시간 = 출근 시간 + 일평균 필요 근무시간 + 점심 1시간

2. 점심 보정:
   - 출근 ≤ 12:00 → 점심 1시간 추가
   - 출근 ≥ 13:00 → 점심 추가 안 함
   - 12:00 < 출근 < 13:00 → 13:00을 기준으로 계산 (출근 = 13:00 취급)

3. 코어타임 보정:
   - 계산된 퇴근 시간 < 16:00 → 16:00으로 올림

4. 현재 시간 ≥ 퇴근 시간 → "지금 퇴근 가능!" 표시
```

### 예시

| 출근 | 일평균 | 점심 | 계산 | 코어타임 보정 | 결과 |
|------|--------|------|------|:------------:|------|
| 09:53 | 7h 39m | +1h | 18:32 | - | 18:32 |
| 10:00 | 5h 00m | +1h | 16:00 | - | 16:00 |
| 09:00 | 4h 30m | +1h | 14:30 | 16:00 | 16:00 |
| 13:30 | 7h 39m | 없음 | 21:09 | - | 21:09 |

## 엣지케이스

- [x] 출근 시간을 읽지 못했을 때 → "출근 기록이 없습니다" 표시
- [x] 이미 퇴근 가능 시간이 지났을 때 → "지금 퇴근 가능!" 초록색 표시
- [x] 코어타임 전에 필요시간 채워질 때 → 16:00으로 올림
- [x] 12:00~13:00 사이 출근 → 13:00 출근으로 취급
- [ ] 금요일 재택 제외 ON일 때 → 금요일은 8h 고정이므로 일평균 대신 8h로 계산
- [ ] 남은 근무일 0일 (월말 완료) → 퇴근 시간 영역 숨김

## 제약사항
- 사용할 라이브러리/컴포넌트: 기존 코드만 확장 (새 파일 없음)
- Out of Scope:
  - 점심시간 커스터마이징 (12~13시 고정)
  - 코어타임 커스터마이징 (10~16시 고정)
  - 반차 출근/퇴근 시간 별도 처리
  - 휴게 시간 (15:09:47 타이머) 차감

## 기술적 리스크

- [x] "출근 09:53" 텍스트 DOM 위치 — 스크린샷으로 확인 완료. "출퇴근" 위젯 내 "출근 HH:MM" 패턴
